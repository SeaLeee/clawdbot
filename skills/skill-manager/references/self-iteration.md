# 自迭代策略指南

## 概述

Skill Manager具备自迭代能力，可以分析自身和其他skill的表现，自动提出并实施改进。

## 自迭代触发条件

### 主动触发
- 用户明确要求更新/改进某个skill
- 用户提供使用反馈指出skill的不足
- 定期自检（通过 `manage_skills.py self-update`）

### 被动触发
- skill执行失败时，分析失败原因并提出改进
- 安全扫描发现问题后，自动改写
- ClawHub有新版本可用时

## 自迭代流程

### 1. 状态分析

```
收集当前skill信息:
  - SKILL.md内容和结构
  - scripts/的功能覆盖度
  - references/的完整度
  - 与其他skill的依赖关系
  - 历史变更记录
```

### 2. 改进识别

常见改进方向：
- **描述优化** — description字段不够清晰或完整
- **工作流补充** — 缺少常见用例的处理流程
- **脚本增强** — 错误处理不够、缺少边界情况处理
- **安全加固** — 扫描发现新的安全规则需要适配
- **依赖更新** — 依赖的工具或API有更新

### 3. 差异生成

生成改进差异（diff）：
- 仅改动必要部分，不重写整个文件
- 每次改动附带理由说明
- 保持向后兼容

### 4. 预览与确认

- 展示改动差异给用户审核
- 说明改动的理由和预期效果
- 用户确认后才应用

### 5. 应用与验证

- 应用改动到文件
- 重新运行安全扫描
- 记录变更日志

## 自迭代原则

1. **最小改动原则** — 每次只改必要的部分
2. **安全优先** — 任何改动后必须通过安全扫描
3. **可回滚** — 保留改动前的备份
4. **透明度** — 所有改动记录在changelog中
5. **用户控制** — 重大改动需用户确认

## Skill Manager自身迭代

Skill Manager可以更新自身的：
- SKILL.md（改进描述和工作流）
- scripts/（修复bug、增加功能）
- references/（更新指南）
- assets/dashboard/（改进UI）

自身迭代步骤：
1. 运行 `manage_skills.py self-update`
2. 先尝试从ClawHub获取更新
3. 若无远程更新，进行自检分析
4. 展示改进建议
5. 用户确认后实施
